# Written Report Outline / Checklist

A structure check to run before submission — not a substitute for writing
the report yourself. Each item below maps to something the rubric or
assignment instructions explicitly asks for.

- [ ] **Introduction** — problem statement, mission alignment, justified
      with cited evidence (not just asserted)
- [ ] **Literature Review** — 10+ scholarly sources, IEEE in-text citations,
      sources critically compared against each other (not just summarized
      one after another), with gaps identified that your project addresses
- [ ] **Methodology** — both datasets described and justified, every
      preprocessing/feature engineering decision explained (missing values,
      county-name standardization, lag window choice, target definition),
      model architectures, experimental design
- [ ] **Results** — experiment table included and referenced in the text,
      figures (learning curves, confusion matrices, ROC curves) labeled and
      discussed, not just pasted in
- [ ] **Discussion** — error analysis tied to specific evidence (not "the
      model overfit" without showing it), bias-variance discussion, honest
      dataset limitations (geographic name mismatches, ACLED reporting gaps,
      the 2017+ time restriction), comparison back to the literature review
- [ ] **Conclusion** — summary of findings, explicit tie back to the mission
      (early-warning potential for peace/humanitarian response), future work
- [ ] **References** — IEEE format, complete reference list
- [ ] Links to GitHub repository and demo video included in the report
- [ ] Word count: 3,500–5,000 words (excluding references, captions, appendices)
- [ ] Written as prose throughout — no excessive bullet points (this
      checklist is an exception, your report isn't)
- [ ] Read it once for your own voice — does it sound like you explaining
      your reasoning, or like a summary of what was done? The rubric and
      AI-detection threshold are both checking for the former.
